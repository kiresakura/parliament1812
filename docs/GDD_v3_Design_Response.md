## 0. 總裁決：評測說對了什麼、漏了什麼

**評測的核心診斷成立，且我方數學覆核無誤**（見 1.1）。一句話總結病灶：

> 聲望「一幣三用」（分數＝票權＝行動費），而情報沒有任何兌換出口。
> 於是這個「資訊戰」遊戲裡，**做資訊戰的人在補貼不做資訊戰的人**。

v3 的設計主軸因此只有一條：**拆掉「一幣三用」，改為「雙幣分權」。**

| 資本 | 載體 | 用途 |
|------|------|------|
| **政治資本** | 聲望 | 票權 + 行動費（不再直接等於分數——計分折半） |
| **情報資本** | 把柄（新資源） | 壓票 / 爆料彈藥 / 揭穿謊言 / 局末計分 |
| **分數** | 兩層制 | 完成議程者永遠排在未完成者之前；聲望折半入分 |

裁決總表：

| 評測意見 | 裁決 | 備註 |
|----------|------|------|
| 罪一：演說掛機最優解 | ✅ 完全採納方案一 | 數學覆核成立（1.1） |
| 罪二：反駁死牌、爆料慈善 | ✅ 採納 + 微調 | 爆料加「輿論紅利 +5」，從慈善變收割（1.2） |
| 罪三：議程失衡、勝負精神分裂 | ✅ 採納 + 1 處替代設計 | 政治殺手不改數值——直接刪，換「黑函作者」（1.3） |
| 罪四：社交層無扳機 | ✅ 完全採納方案二 | 表態語義精確化（1.4） |
| 罪五：決策量太少 | ✅ 採納 + 補強 | 評測漏了 R1 盲射的根因：法案最後才公布。v3 改為第一幕開頭公布（1.5） |
| 課金罪一：strictly-better 卡 | ✅ 無條件採納 | 議長裁決刪除；給出 sidegrade 重設計範本（2.1） |
| 課金罪二：賣排面沒舞台 | ✅ 採納 | 優先序微調：先做翻牌特效與議會紀錄版式（搭便車於既有動畫元件）（2.2） |
| 課金罪三：商店開門客人走光 | ✅ 採納 + 重排 | 盤點發現後端 migration（季票/週任務/推薦獎勵）**已存在**，提前成本比評測想的低（2.3） |
| 課金罪四：模型不閉環 | ✅ 採納 | 補上評測要求的數字目標（2.4） |
| 附錄 8 條規則 Bug | ✅ 全修 | 逐條裁決見第 5 節 |

**評測漏掉、v3 補上的四件事：**

1. **R1 盲射的根因修法**：法案改到第一幕「最先」公布（原設計刻意最後公布），讓私訊與第一回合質詢有的放矢。
2. **把柄防刷上限**：同一持有者對同一目標**最多 1 枚**把柄（評測未規定，否則把柄變成可對單一受害者無限開採的礦）。
3. **挑戰潮（challenge-spam）反向風險**：演說被砍後 meta 可能擺向全員質詢。反質詢改版（擋下可反看對方議程類別）本身就是煞車，但模擬驗收必須加一條「質詢率 < 45%／回合」。
4. **斷線/掛機的二階效應**：謊言開始有 -15 代價之後，被抓包的人會直接拔線。需要配套：超時=棄權、連 2 次超時 AI 接管＋本局不計排位。

## 1. 玩法五宗罪：逐條深度分析與 v3 規格

### 1.1 罪一【演說掛機】— 診斷覆核與修法

**覆核評測數學（GDD v2 原始數值，4 人局）：**

- 掛機仔（保守黨）：60 + 演說×3(+30) = 90 聲望 → 票數 1+⌊90/20⌋ = **5 票**；議程失敗仍得 90 分，沒被質詢再 +10（金鐘罩）= 100。
- 教科書玩家（自由派）：50 −5(質詢) +15(結盟) −10(爆料) = 50 聲望 → **3 票**；議程成功 40+50+15 = 105，失敗 = 65。

結論成立：認真玩只在最好情況下贏 5 分，最壞情況輸 35 分，**而且掛機仔票更多、自己議程通過率反而更高**。超時自動出演說 = 系統替掛網者執行最優策略。這不是平衡問題，是經濟結構問題。

**v3 修法（採納方案一全部七條，數值定稿如下）：**

#### A. 新資源「把柄」（Leverage）

| 規格 | 值 |
|------|-----|
| 取得 | 質詢成功 → 獲得目標把柄 ×1 |
| 可見性 | **持有數公開，內容私密**（全桌看得到「他手上有 2 枚把柄」，不知道是誰的） |
| 上限 | 同一持有者對同一目標最多 1 枚（v3 新增，防單一目標開採） |
| 出口 1：壓票 | 表決階段打出 → 目標本次投票 **−2 票**（目標保底 1 票；每人對同一目標限用 1 枚/局） |
| 出口 2：囤積 | 局末每枚未用把柄 **+10 分** |
| 出口 3：彈藥 | 爆料改為消耗 1 枚對應目標的把柄（見 1.2） |
| 出口 4：揭穿 | 局末結算：你持有 X 的把柄，且 X 最終投票 ≠ X 最後一次公開表態 → X **−15 分**、你 **+5 分**（MVP 自動結算；完整版做唱票現場揭穿） |
| 失效 | 目標倒戈 → 指向他的所有把柄銷毀，每位持有者退 **+5 聲望** + 私密通知「情報已過期」 |

設計註：把柄是「**祕密的鈔票化**」。質詢期望值變為：EV ≈ p(未被擋) × (+10 分或更高的選擇權) − 5 聲望費。以 20% 反駁率估算 ≈ +3 起跳、帶選擇權溢價——終於和演說打平且更有趣。

#### B. 演說遞減 + 強制表態

| 規格 | 值 |
|------|-----|
| 聲望收益 | 第 1 次 +10 → 第 2 次 +5 → 第 3 次起 +0 |
| 強制表態 | 每次演說**必須公開宣告**支持 A/B/C 之一（可以說謊），記入公開「言行紀錄」 |
| 說謊代價 | 由把柄出口 4（揭穿）執行：−15 分 |
| 超時預設 | 自動出演說 → 改為**棄權**（+0、無表態、不出牌）。連續 2 次超時：標記 AFK，AI 接管，本局不計排位獎勵 |

設計註：演說從「免費印鈔機」變成「遞減收益 + 留下可被攻擊的言論紀錄」。掛機者要嘛 0 收益（棄權），要嘛留下把柄持有者可收割的謊言。

#### C. 反駁 → 反質詢

| 規格 | 值 |
|------|-----|
| 擋下質詢 | +5 聲望，**並得知質詢者的議程類別**（法案派/條件派/攪局派）——反偵察 |
| 無人質詢 | +3 聲望保底（死牌消失） |

設計註：這張牌同時是質詢潮的煞車——質詢撲空不只 −5，還洩漏自己的類別情報。

#### D. 倒戈付費化

| 規格 | 值 |
|------|-----|
| 成本 | −10 聲望 |
| 連動 | 銷毀所有指向自己的把柄，持有者各退 +5 聲望 + 私密「情報過期」通知 |
| 判定 | 雙面人/影子大師等「被知道」判定，一律以**局末當前議程**為準（修附錄 Bug 6） |

#### E. 計分兩層制

```
最終分 = 議程分（完成 = 滿分 / 未完成 = 0）
       + ⌊剩餘聲望 / 2⌋          ← 折半，聲望不再是分數本體
       + 把柄分（每枚未用 +10）
       + 表態結算（指控/擔保/揭穿，見 1.4）
       + 社交成就分

排名 = 先分層（完成議程 > 未完成），層內按最終分。
```

掛機仔的 90 聲望折半剩 45 分、無議程、無把柄、無表態收益，**永遠排在任何完成議程者之後**。

#### F. 回合 3 → 4

時間軸定稿（目標 9.5–11 分鐘，統一兩份文件的時長口徑，修附錄 Bug 4）：

```
第一幕 暗流 90s   ：法案公布（最先！）→ 派系/議程/手牌 → 私訊 2 條
第二幕 辯論 ×4 回合：選牌 30s → 翻牌結算 15s → 公開聊天 40s（每回合 85s，共 340s）
第三幕 表決 90s   ：回顧 15s → 投票+壓票 45s → 唱票 30s → 議程揭示
```

手牌 5 張、出 4 張、剩 1 張。第 4 回合只剩 2 選 1，選擇變薄——標記為 Beta 調參項（候選：起手 6 張）。

**罪一驗收門檻（模擬器跑 1,000 局，4 人局）：**
- 純演說策略勝率（進前 1 名比例）**< 25%**
- 質詢出現率 **< 45%/回合**（防 meta 反轉成全員 aggro）

### 1.2 罪二【反駁死牌、爆料慈善】

反駁已由 1.1-C 解決。爆料的搭便車困境，v3 修法：

| 爆料 v3 | 值 |
|------|-----|
| 前提 | 持有目標的把柄（取代「曾質詢/結盟過」——統一到把柄系統） |
| 成本 | 消耗該枚把柄，**自身聲望費歸零** |
| 收益 | 目標 −20 聲望 + 議程公開 + 你 **+5 聲望（輿論紅利）** |
| 演出 | 「號外！」全場停格時刻（MVP：5 秒頭條 overlay + 被爆者 10 秒申辯；Beta：15 秒報紙動畫 + 付費版面模板） |

經濟帳：舊版爆料淨投資 −15 換全桌受益；v3 是「花 5（質詢）→ 賺 5（紅利）+ 拔掉一個對手的偽裝 + 真相使者成就 +10」。**爆料從公共財變成私人收割，順便是全遊戲最大的可錄屏舞台。**

注意保留的張力：爆料消耗把柄 = 放棄該枚的 +10 囤積分與壓票權。打不打、何時打，仍是真決策。

**驗收門檻：爆料每局出現率 > 35%。**

### 1.3 罪三【議程失衡】— 議程表 v3

| 議程 | v2 | v3 | 變更理由 |
|------|-----|-----|---------|
| 法案擁護者 A/B/C | 40 | **50** | 主線玩法不該是最低薪 |
| 穩健派 A/B | 55 | 55 | 不變 |
| 人望王 | 50 | 50 | 不變 |
| 合縱連橫 | 50 | 50 | 明確定義：與 **2 個不同對象**成功結盟（修 Bug 3） |
| 議會縱火犯 | 60 | **55 + 關鍵票條件** | 條件：存在至少一種你的改投方式，使「票差 ≤3 / 流局」不再成立——你必須是**主動維持混亂**的人。伺服器以反事實重算驗證（≤4 種改投 × 重新計票，成本極低） |
| 雙面人 | 65 | **45 + 誘騙條件** | 條件改為「至少被質詢成功 1 次，且局末議程仍未被正確得知」。從躲貓貓變成主動誘騙（配合偽造文書/倒戈），獎勵給「玩得最騷的人」而不是「拒絕玩的人」 |
| 政治殺手 | 60 | **刪除** | 玩家對達成條件幾乎沒有代理權，改數值救不了 |
| 黑函作者（新） | — | **50** | 局末同時持有 ≥2 名**不同玩家**的把柄。完全自主可達成、餵養質詢經濟、主題滿分（勒索者）。注意：把柄花掉就不算——囤積 vs 使用的兩難由議程放大 |

**成就修正：** 真相使者 = 成功爆料 **1** 人 +10（修 Bug 1：牌庫只有 1 張爆料，原條件數學上不可能）；影子大師與雙面人判定一律以局末當前議程為準。

**依賴關係警告（開發順序硬約束）：** 兩層計分制（1.1-E）**不可以**先於議程重平衡上線——否則「完成議程者優先」會讓躺著過的舊縱火犯/雙面人霸榜，比現狀更糟。Sprint A 內兩者必須同包發布。

### 1.4 罪四【社交層無扳機】— 採納方案二，語義定稿

1. **全程外交**：第一幕私訊 2 條；之後**每回合聊天階段 1 條**。中盤才可能「對 A 真話、對 B 假話」，單排玩家獲得協商管道。
2. **一鍵表態**（每聊天階段限用 1 次，免打字，手機友善）：
   - **指控**：「X 是攪局派」→ 局末揭示：對 **+8 分** / 錯 **−3 分**（4 人局攪局派基率 ~25%，盲猜期望值約 −0.25，必須靠情報才划算——這正是情報的又一個出口）。
   - **擔保**：「我以聲望為 X 作保（X 言行一致）」→ X 最終投票與其最後公開表態一致 **+5 分** / 不一致 **−15 分**。擔保錯的重罰，就是「被騙」的機制化呈現。
   - 所有表態進公開言行紀錄，並印入賽後議會紀錄（分享素材 + 課金版面掛載點）。
3. **盟約提案 handshake**：聊天階段可私下對 1 人發提案；下回合雙方都出結盟才成立。**同一對象重複結盟收益遞減 +15 → +5**——雙排語音印分機報廢。
4. **號外時刻**：見 1.2。
5. **唱票終幕**：表決改為按聲望**由低到高依序公開亮票**（每人 2 秒、自動推進）。最後一位高票權玩家亮票前的全場屏息 = 造王者時刻。MVP 做簡版（依序亮票），完整版加把柄現場揭穿插斷。
6. **AI 透明化**：AI 席位明確標識；新手前 5 場進保護房（只配新手/AI）。在說謊遊戲裡讓老玩家對不會說謊的機器人演戲，是對核心樂趣的自殺。

**驗收門檻：表態使用率 > 60%/局；單排 vs 雙排勝率差 < 8%；「再來一局」點擊率 > 50%。**

### 1.5 罪五【決策量太少】— 採納 + 根因補強

決策量帳本（v2 → v3）：

| 決策點 | v2 | v3 |
|--------|----|----|
| 出牌（含目標指定） | 3 | 4 |
| 表態（指控/擔保） | 0 | 最多 4 |
| 私訊/盟約提案 | 2（僅第一幕） | 2 + 4 |
| 壓票（表決階段） | 0 | 0–2 |
| **合計** | **~5** | **~12–16** |

決策密度 ×3，單局時長只 +約 1 分鐘。「讀出牌模式」在 4 回合 + 表態紀錄 + 言行紀錄之下才有樣本量。

**評測沒給的根因修法——R1 盲射：** v2 刻意把法案放在第一幕最後公布（製造不確定性），代價是私訊階段與第 1 回合全是擲骰子。v3 把**法案公布移到第一幕第一步**：私訊立刻有實質內容（「你支持 B 嗎？」），R1 質詢有了瞄準依據。不確定性的來源交給隱藏議程承擔——這本來就是它的工作。

**內容見底問題**：重玩性的主驅動是議程池而非法案數——議程決定行為劇本，法案只是舞台佈景。Beta 目標：法案 3 部、**議程池 ≥ 12**（v3 表 9 個 + 3 個條件派變體）。偽造文書與倒戈本身就是劇本變異器。

### 1.6 派系平衡（修附錄 Bug 2、8）

「聲望=分數=票權」體系下 60/45 起始差 = 開局送 15 分 + 1 票，且「商行有資金系統補償」是全文唯一出現、無定義的幽靈系統。v3：

- **「資金系統」字眼刪除**（Bug 2 結案——選「刪」不選「做」，MVP 不養幽靈）。
- 起始聲望維持 60/50/45（派系個性保留），但補償改為**情報資產**（on-theme：情報派系拿情報，不拿聲望）：
  - 自由派：第一幕私訊額度 2 → **3 條**。
  - 商行聯盟：開局獲得 1 名隨機玩家的**議程類別**情報（私密）。
- 計分折半後，起始差距在分數端縮為 7.5 分；票權端 4:3:3 的開局差留給模擬器驗證（驗收：各派系勝率差 < 6%）。

## 2. 課金四宗罪：逐條裁決

### 2.1 稀有度毒藥（無條件採納，含重設計範本）

**設計鐵律入文：稀有度 = sidegrade（同強度、不同形狀），永不違反同時出牌支柱。**

| 卡 | v2（違規） | v3 重設計 |
|----|-----------|----------|
| 議長裁決（傳奇） | 看完所有人的牌再決定 → **直接廢掉核心支柱** | **刪除。** 替代傳奇「議事規則」：你出的牌改變自己的結算優先級（你仍然盲出——改的是結算順序，不是資訊） |
| 質詢·深挖（精良） | 質詢 + 偷看 1 張手牌 → strictly better | 「交叉詰問」：看議程 + 1 張手牌，**但目標得知是你質詢的並反看你 1 張手牌**（資訊對洩，換型不換強度） |
| 演說·民心所向（史詩） | 條件 +25 → 純數值膨脹 | +10；若你全場聲望最低改為 +15，**但須公開你的派系**（收益換資訊代價） |

## 7. 技術棧決議：回歸 Flutter

### 7.1 決定

| 層 | 技術 |
|-----|------|
| 客戶端 | **Flutter 3.x + Riverpod**（`app/`，既有 89 個 dart 檔） |
| 後端 | Rust（Axum + Tokio）——不變，權威伺服器 |
| 即時通訊 | WebSocket（tokio-tungstenite ↔ web_socket_channel）——不變 |
| 資料庫/快取/部署 | PostgreSQL / Redis / Fly.io——不變 |
| Godot 客戶端 | `godot-client/` **封存**：停止開發、保留參考，不刪除 |

### 7.2 理由（按權重排序）

1. **聊天是核心機制，輸入法是核心依賴。** 本作的社交層全靠文字（公屏/私訊/表態），目標市場是台灣（LINE 分享鏈）。Flutter 的中文 IME、輸入框、剪貼簿行為是原生級；Godot 的行動端文字輸入與 IME 整合長期是弱項。對一款「打字說謊」的遊戲，這一條就足以定案。
2. **既有資產不對稱。** `app/` 已有：OAuth（Google/Apple）、i18n（zh/en）、WebSocket 服務、freezed 模型、任務/成就/好友/排行/圖鑑畫面、出牌/翻牌/計票/排名四套動畫元件。`godot-client/` 45 個腳本要重建以上全部。回歸 Flutter 是撿回資產，不是沉沒成本謬誤——資產還活著且和後端協議對齊。
3. **本作是 UI 密集型卡牌遊戲，不是物理/3D 遊戲。** 手牌、頭像、聊天、表決面板——全是 Flutter 的主場。需要的「遊戲感」集中在少數高光時刻（翻牌、號外、唱票），用 `flutter_animate` + **Rive**（向量動畫、檔案小、可程式控制）覆蓋足夠。
4. **單一語言鏈降低一人團隊的切換成本。** Dart（客戶端）+ Rust（伺服器），不再維護第三套 GDScript 心智模型。

### 7.3 風險與緩解

| 風險 | 緩解 |
|------|------|
| 翻牌/號外特效品質不如遊戲引擎 | Rive 資產管線；P0 只做 4 個高光時刻，不做全域粒子 |
| Flutter Web 首載體積（若做 Web 版） | Web 降為 P2；屆時用 CanvasKit + deferred loading 調優 |
| Godot 期間累積的素材（assets/characters、music） | 為通用素材（PNG/OGG），Flutter 直接取用 |

### 7.4 配套修訂

- `CLAUDE.md`：技術棧、專案結構、單人模式檔案路徑全面校正（指向 `app/` 實際結構；`platforms/flutter/` 路徑早已不存在）。
- `docs/GDD_v2.md` §12.1：客戶端列改為 Flutter；文件頭加註 v3 修訂指引。

## 8. 結語：v3 之後，這遊戲獎勵什麼？

v2 的回答是「獎勵聲望最高的人」，而聲望最高的人是不玩遊戲的人。
v3 的回答：**獎勵知道得最多、說得最準、騙得最久的人**——

- 知道得最多：把柄四出口，每一條情報都有價格標籤。
- 說得最準：指控/擔保把嘴砲變成押注，命中者拿走說錯者的籌碼。
- 騙得最久：雙面人 45 分的誘騙條件、偽造文書反制爆料、倒戈燒檔案——說謊有成本，但騙到底有冠軍。

評測最後一句說得對：核心創意值得做。v3 的工作就是讓經濟學站到創意這一邊。先跑 Sprint A 的 1,000 局模擬，數字不會說謊——這點和玩家不一樣。
