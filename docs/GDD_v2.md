## 1. 概覽

### 1.1 遊戲名稱

**1812 國會風雲**（1812: Parliament Storm）

### 1.2 一句話描述

> **資訊戰 × 策略卡牌 × 社交推理** — 同時出牌、暗中試探、當面撒謊，在 10 分鐘的議會中完成你的隱藏議程。

### 1.3 核心賣點

| # | 賣點 | 說明 |
|---|------|------|
| 1 | **同時出牌零等待** | 所有人同時決策、同時翻牌——沒有「等別人操作」的無聊時間。每次翻牌都是腎上腺素時刻。 |
| 2 | **資訊就是武器** | 核心資源不是數值，是「誰知道什麼」。質詢偷看底牌、爆料公開秘密、結盟共享情報——每張牌都在操控資訊流。 |
| 3 | **說謊是核心技能** | 被質詢後可以對所有人宣稱假議程。讀人、測謊、虛張聲勢是贏的關鍵。 |
| 4 | **10 分鐘一局** | 三幕劇結構：暗流（90秒）→ 辯論（6分鐘）→ 表決（90秒）。午休能打三局。 |
| 5 | **雙模式** | 單機 Roguelike 爬屆構築（一個人也好玩）+ 多人資訊戰（社交爆發）。 |

### 1.4 對標競品

| 競品 | 我們學什麼 | 我們不學什麼 |
|------|-----------|-------------|
| **Coup / Resistance** | 隱藏身份 + bluff 的核心張力 | 太簡單、缺乏策略深度 |
| **Blood on the Clocktower** | 豐富的資訊生態 + 社交推理深度 | 需要主持人、學習門檻高、時間長 |
| **Slay the Spire** | 牌組構築 + Roguelike 的長線留存 | 純單機、無社交元素 |
| **Among Us** | 社交推理的病毒傳播力 + 好友房 | 缺乏策略深度、重複性高 |

**1812 的交叉定位：** Coup 的 bluff 張力 × STS 的卡牌深度 × Among Us 的社交傳播力。市場上不存在這個組合。

### 1.5 目標受眾

- **核心族群：** 20-35 歲，玩過桌遊（Coup / Avalon / 狼人殺）或社交推理手遊
- **策略玩家：** 追求卡牌 Combo、構築深度、數值優化
- **社交推理玩家：** 追求心理博弈、嘴砲、讀人的快感
- **歷史愛好者：** 對 19 世紀英國政治有興趣（加分項，非必要）

## 2. 設計哲學

### 2.1 v1 的問題

| 問題 | 原因 | 後果 |
|------|------|------|
| 辯論 = RPG 戰鬥 | 質詢是攻擊、反駁是防禦，只是換了政治皮 | 失去社交推理的核心——資訊博弈 |
| 75% 時間在等待 | 回合制輪流行動 | 3 人在等 1 人操作，節奏拖沓 |
| 聲望是生命值 | 歸零 = 政治死亡 | 被集火的人體驗極差，缺乏反制手段 |
| 社交層是附加品 | 聊天和出牌分離 | 嘴砲不影響遊戲結果，社交推理淪為裝飾 |

### 2.2 v2 的設計原則

> **原則一：每張牌都是政治訊號。**
> 你對誰出了質詢，所有人都看到了。這會改變他們對你立場的判斷。出牌不只有機械效果，更有社交效果。

> **原則二：資訊是最稀缺的資源。**
> 聲望可以恢復，但洩漏的情報收不回來。遊戲的核心張力來自「你知道多少」和「別人知道你知道多少」。

> **原則三：零等待時間。**
> 所有人同時出牌、同時翻牌。沒有任何玩家在等別人操作。整局 10 分鐘裡每一秒都在做決策或社交互動。

> **原則四：說謊和讀人是核心技能。**
> 被質詢後，你可以在公開聊天裡說「我的議程是 B」——但你可能在騙人。遊戲獎勵社交能力，而非單純的卡牌數值計算。

## 3. 遊戲模式

### 3.1 多人模式：議會資訊戰（核心模式）

**概念：** 4-6 人進入一場議會，各自持有隱藏議程。透過同時出牌的辯論回合，試探、欺騙、結盟，最終推動自己支持的法案版本通過。**贏家不是聲望最高的人，是完成隱藏議程的人。**

| 屬性 | 值 |
|------|-----|
| 玩家人數 | 4 人（MVP）/ 6 人 / 8 人 |
| 單局時長 | 10-12 分鐘 |
| 結構 | 三幕劇（暗流 → 辯論 → 表決） |
| 核心動詞 | 偵查 / 隱藏 / 結盟 / 背叛 |
| 勝利條件 | 完成隱藏議程 |

### 3.2 單機模式：國會 Roguelike

**概念：** 類似 STS 的爬塔構築，但「塔」是一屆又一屆的國會。保留 v1 的設計框架，但核心機制換用 v2 的卡牌系統和四維度系統。

> 詳見 [Section 9](#9-單機模式國會-roguelike)。

## 4. 核心機制：多人模式

### 4.1 三幕劇結構

```
┌─────────────────────────────────────────────────────────────┐
│                      一局遊戲（10-12 分鐘）                   │
│                                                             │
│  ┌───────────┐   ┌─────────────────────┐   ┌───────────┐   │
│  │ 第一幕     │   │ 第二幕               │   │ 第三幕     │   │
│  │ 暗流       │──▶│ 辯論                 │──▶│ 表決       │   │
│  │ 90 秒      │   │ 6 分鐘（3 回合）      │   │ 90 秒      │   │
│  │            │   │                     │   │            │   │
│  │ • 抽手牌    │   │ 每回合：             │   │ • 投票 ABC  │   │
│  │ • 收議程卡  │   │ ① 同時選牌（30秒）   │   │ • 聲望加權  │   │
│  │ • 2 條私訊  │   │ ② 同時翻牌           │   │ • 議程揭示  │   │
│  │            │   │ ③ 效果結算           │   │ • 勝負判定  │   │
│  │            │   │ ④ 公開聊天（45秒）   │   │            │   │
│  └───────────┘   └─────────────────────┘   └───────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 第一幕：暗流（Undercurrent）— 90 秒

**目的：** 建立初始資訊不對稱，讓玩家開始猜測彼此立場。

| 步驟 | 動作 | 說明 |
|------|------|------|
| 1 | 派發角色 | 隨機分配派系角色（保守黨/自由派/商行聯盟） |
| 2 | 派發議程 | 每人收到 1 張隱藏議程卡（勝利條件） |
| 3 | 抽手牌 | 從個人牌庫抽 5 張手牌 |
| 4 | 私訊階段 | 可發送 2 條私訊給任意玩家。私訊內容只有收發雙方看到。 |
| 5 | 法案公布 | 公開本局法案及 A/B/C 三個版本 |

**設計意圖：**
- 私訊限制為 2 條，迫使玩家做選擇——你想試探誰？想拉攏誰？
- 收到私訊本身就是情報：「他主動來找我，說明他可能需要我的票」
- 法案最後公布，這樣私訊階段玩家只能根據派系猜測立場，增加不確定性

### 4.3 第二幕：辯論（Debate）— 3 回合，每回合約 2 分鐘

**這是遊戲的核心。** 三個回合，每回合四步：

#### 步驟 ① 選牌（30 秒）

所有人**同時**從手牌中選擇 1 張卡牌。需要指定目標的卡牌（質詢、爆料等）在此時選定目標。選牌是暗蓋的，其他人看不到你選了什麼。

**UI 交互：** 手牌水平排列在螢幕下方。點選一張卡牌後上滑鎖定。需要指定目標的卡牌會彈出玩家頭像列表。倒計時結束前未選牌則自動出「演說」（最安全的預設）。

#### 步驟 ② 翻牌（同時揭示）

所有人的牌**同時翻開**。這是遊戲中最緊張的時刻——你不知道別人打了什麼，也不知道有沒有人對你出了質詢。

**視覺設計：** 牌面從暗色翻轉為亮色，配合音效和震動反饋。如果有人對你出牌，你的頭像會有紅色警示框。

#### 步驟 ③ 效果結算

卡牌效果按照**優先級順序**結算：

| 優先級 | 卡牌類型 | 結算規則 |
|--------|---------|---------|
| 1（最先） | 🛡️ 反駁 | 先結算所有反駁。成功擋住質詢的玩家 +5 聲望。 |
| 2 | 🗡️ 質詢 | 檢查目標是否出了反駁。未被擋住的質詢：目標私下向質詢者透露議程。花費 5 聲望。 |
| 3 | 💣 爆料 | 質詢者已知目標議程時才能使用。公開揭露目標議程。花費 10 聲望，目標 -20。 |
| 4 | 🤝 結盟 | 檢查目標是否也對你出了結盟。雙向成功：雙方各 +15 聲望，共享議程。單向失敗：-5 聲望。 |
| 5 | 📢 演說 | 無條件 +10 聲望。 |
| 6（最後） | 🔄 倒戈 | 秘密替換議程卡。無人知曉。 |

**關鍵互動規則：**

- **質詢 vs 反駁**：如果你質詢的目標出了反駁，質詢失敗，你仍花費 5 聲望但得不到情報。反駁者 +5 聲望。
- **質詢 vs 質詢**：如果兩人互相質詢，雙方都成功——各自私下看到對方議程。但其他人看到你們互相質詢，會推測你們是敵對關係。
- **結盟 vs 結盟**：雙向成功，雙方成為盟友，共享議程資訊，各 +15 聲望。**但盟約不是永久的——下回合任何一方可以出爆料背叛。**
- **爆料的前提**：你必須已經通過質詢或結盟知道目標的議程，才能打爆料。系統會驗證這一點。
- **倒戈的隱蔽性**：倒戈結算時，其他人只看到你出了一張牌（背面朝上），不知道你打了什麼。但這本身就是情報——「他出了一張沒有效果的牌，他在隱藏什麼？」

#### 步驟 ④ 公開聊天（45 秒）

翻牌結算後，開放全體聊天（文字或語音）。這是社交推理的核心場：

- **控訴：** 「你剛才質詢了我，你是不是在查我的底？你到底是什麼派的？」
- **撒謊：** 「他質詢了我，我的議程是 A，我們是同一邊的！」（可能是假的）
- **結盟協商：** 「下一回合我們互相出結盟？」
- **挑撥離間：** 「你注意到了嗎？他和那個人都出了演說，他們在保存實力，等最後一回合爆發。」

**設計意圖：** 聊天不是無關緊要的附加品——它是遊戲機制的一部分。你在聊天中說的每句話都可能是真的或假的，其他人必須根據你的行為（出牌模式 + 言語 + 前幾回合的歷史）來判斷你是否值得信任。

### 4.4 第三幕：表決（Vote）— 90 秒

#### 投票機制

1. **法案回顧**（15 秒）：顯示法案 A/B/C 三個版本的效果
2. **投票**（45 秒）：每人選擇 A / B / C / 棄權
3. **計票**（30 秒）：聲望加權計票 → 公布結果 → 議程揭示 → 勝負判定

#### 聲望加權計票

```
玩家票數 = 1 + floor(聲望 / 20)

範例：
  聲望 80 的玩家 = 1 + 4 = 5 票
  聲望 40 的玩家 = 1 + 2 = 3 票
  聲望 10 的玩家 = 1 + 0 = 1 票

票數最多的版本通過。平手則比較支持者總聲望。
```

**設計意圖：** 聲望高 = 投票權重大，但不是壓倒性的。聲望 80 vs 聲望 40 是 5:3，不是 80:40。這確保低聲望玩家仍有博弈空間，同時讓聲望管理有意義。

#### 議程揭示與計分

投票結果公布後，所有人的隱藏議程卡翻開。完成議程的玩家獲勝。

> 詳細計分規則見 [Section 10](#10-結局與計分系統)。

### 4.5 聲望系統（重新定義）

**v1 的問題：** 聲望 = 生命值，歸零 = 淘汰。被集火的人體驗極差。

**v2 的重新定義：** 聲望是**三合一資源**——

| 功能 | 說明 | 為什麼重要 |
|------|------|-----------|
| **投票權重** | 聲望越高，投票時票數越多 | 高聲望 = 在表決中更有影響力 |
| **行動成本** | 質詢花 5、爆料花 10 | 積極行動需要「燃燒」政治資本 |
| **計分加成** | 最終分數 = 議程分 + 剩餘聲望 | 鼓勵效率——用最少聲望完成議程 |

**關鍵改變：聲望不會歸零淘汰。** 聲望最低降到 5（保底），確保所有玩家都能參與到最後的投票。低聲望的懲罰是投票權重低 + 最終得分低，而非被踢出遊戲。

#### 初始聲望

| 派系 | 初始聲望 | 說明 |
|------|---------|------|
| 保守黨 | 60 | 穩定起始，防禦風格 |
| 自由派 | 50 | 中等起始，需要靠連段建立聲望 |
| 商行聯盟 | 45 | 最低起始，但有資金系統補償 |
| 無派系（通用角色） | 55 | 平均值 |

## 5. 卡牌系統

### 5.1 設計原則

> **v1 的卡牌：** 法案卡 / 策略卡 / 人脈卡 / 陰謀卡 / 交易卡 — 5 種類型，按功能分類。
>
> **v2 的卡牌：** 質詢 / 反駁 / 演說 / 結盟 / 爆料 / 倒戈 — 6 種類型，按**政治行動**分類。每種牌的核心設計不是「造成 X 傷害」，而是「改變資訊流的方式」。

### 5.2 六種基礎卡牌

#### 🗡️ 質詢（Challenge）

| 屬性 | 值 |
|------|-----|
| 目標 | 指定 1 名玩家 |
| 聲望成本 | -5 |
| 成功效果 | 目標私下告訴你他的議程 |
| 失敗條件 | 目標出了反駁 |
| 社交訊號 | 所有人看到你質詢了誰 → 暴露你的懷疑對象 |

**策略深度：**
- 質詢的情報是私密的——只有你知道。你可以選擇是否（以及如何）分享這個情報。
- 質詢本身是公開的——其他人會推測你和目標的關係。
- 連續質詢同一人可能暴露你的真實意圖。

#### 🛡️ 反駁（Rebuttal）

| 屬性 | 值 |
|------|-----|
| 目標 | 無（自動防禦來自任何人的質詢） |
| 聲望成本 | 0（被觸發時 +5） |
| 成功效果 | 擋住質詢，+5 聲望 |
| 失敗條件 | 沒人質詢你（這張牌浪費了） |
| 社交訊號 | 所有人看到你出了反駁 → 「他在防備什麼？」 |

**策略深度：**
- 反駁是一場賭博——你預判會被質詢才出，但如果沒人來你就浪費了一回合。
- 連續出反駁會被視為「有重要情報要藏」，反而吸引更多質詢。
- 高手的反駁時機選擇反映了對其他玩家行為的預判能力。

#### 📢 演說（Speech）

| 屬性 | 值 |
|------|-----|
| 目標 | 無 |
| 聲望成本 | 0（獲得 +10） |
| 效果 | 安全地增加聲望 |
| 社交訊號 | 「他在積蓄力量」或「他沒有想打的目標」 |

**策略深度：**
- 最安全的選擇，但零情報收益。
- 演說 = 在資訊戰中按兵不動。有時候這是對的（你已經有足夠情報），有時候是浪費（你什麼都不知道卻不去查）。
- 連續演說的人要嘛是高手（已經知道所有情報，在攢聲望等最後投票），要嘛是新手（不知道該幹嘛）。

#### 🤝 結盟（Pact）

| 屬性 | 值 |
|------|-----|
| 目標 | 指定 1 名玩家 |
| 雙向成功 | 雙方各 +15 聲望，共享議程資訊 |
| 單向失敗 | 你 -5 聲望（伸手被拒） |
| 社交訊號 | 所有人看到你想和誰結盟 |

**策略深度：**
- 結盟是信任的終極考驗。你和對方都出結盟才能成功。
- 成功結盟的收益極高（+15 聲望 + 互看議程），但失敗的代價也不小（-5 聲望 + 暴露了你想拉攏誰）。
- **結盟後的背叛**是遊戲中最戲劇性的時刻：你和對方結盟後互看了議程，下一回合你直接出爆料把他的議程公諸於世。這在機制上完全合法，但社交代價極大。
- 這製造了「囚徒困境」——結盟帶來雙贏，但單方面背叛帶來更大優勢。

#### 💣 爆料（Exposé）

| 屬性 | 值 |
|------|-----|
| 目標 | 指定 1 名你已知其議程的玩家 |
| 聲望成本 | -10（你）/ -20（目標） |
| 效果 | 向所有人公開目標的議程 |
| 前提條件 | 你必須已透過質詢或結盟知道目標議程 |
| 社交訊號 | 核武器級別的訊號——你在公開宣戰 |

**策略深度：**
- 爆料是兩步走策略：先質詢拿情報，再爆料公開處刑。
- 被爆料的人不只損失聲望——更關鍵的是所有人都知道了他的議程，投票時可以針對性地聯手阻止他。
- 爆料者的代價也很大：-10 聲望，而且暴露了自己「花了資源在調查這個人」，等於間接透露自己和目標立場對立。
- 爆料的最佳時機是第三回合——最後翻牌時公開敵人議程，讓其他人在投票時知道該防誰。

#### 🔄 倒戈（Defection）

| 屬性 | 值 |
|------|-----|
| 目標 | 無（對自己使用） |
| 聲望成本 | 0 |
| 效果 | 從備選議程池中抽 1 張替換當前議程 |
| 社交訊號 | 其他人看到你出了一張「未知牌」（不知道是什麼） |

**策略深度：**
- 倒戈是整個遊戲中最詭異的牌。你放棄了已知的議程，賭一個新的。
- 所有之前別人對你的調查全部作廢。如果有人質詢過你知道你的議程，倒戈讓那個情報變成假情報。
- 倒戈的風險是你可能抽到更難完成的議程。
- 倒戈出現在牌庫中的數量極少（每人牌庫最多 1 張），所以不是常規策略。

### 5.3 卡牌稀有度

保留產品定位文件中確定的四級稀有度體系，在 v2 機制中的具體表現：

| 稀有度 | 視覺 | 與基礎版差異 | 範例 |
|--------|------|------------|------|
| ⬜ 普通 | 白框 | 上述基礎效果，無附加 | 質詢：花 5 聲望偷看議程 |
| 🟦 精良 | 藍框 | 基礎效果 + 1 個附加效果 | 質詢·深挖：偷看議程 + 偷看目標 1 張手牌 |
| 🟪 史詩 | 紫框 | 條件觸發的強化效果 | 演說·民心所向：+10 聲望；若你聲望全場最低，改為 +25 |
| 🟨 傳奇 | 金框 | 改變規則的效果 | 議長裁決：本回合你最後翻牌，看完所有人的牌再決定效果和目標 |

**稀有度 = 策略彈性，不是數值膨脹。** 傳奇牌不是「質詢造成雙倍傷害」，而是「質詢的規則被改變了」。

### 5.4 派系專屬牌

每個派系有 2-3 張專屬卡牌，提供獨特的政治能力：

#### 保守黨專屬

| 卡牌 | 效果 | 稀有度 |
|------|------|--------|
| 🏛️ 王室否決 | 本回合你免疫所有質詢和爆料 | 精良 |
| ⛪ 教會庇護 | 選擇 1 名玩家，使其本回合聲望不會因任何效果下降 | 史詩 |

#### 自由派專屬

| 卡牌 | 效果 | 稀有度 |
|------|------|--------|
| 📰 街頭小報 | 隨機揭露 1 名玩家議程的 1 個關鍵詞（不是完整議程） | 精良 |
| 📣 群眾演說 | +15 聲望，並且所有其他玩家各 -3 聲望 | 史詩 |

#### 商行聯盟專屬

| 卡牌 | 效果 | 稀有度 |
|------|------|--------|
| 💰 收買 | 花費 15 聲望，偷看目標議程（不被反駁擋住，且不公開顯示） | 精良 |
| 📋 偽造文書 | 你可以向其他人展示一個假議程（由你選擇內容） | 史詩 |

### 5.5 手牌管理

| 屬性 | 值 |
|------|-----|
| 初始手牌 | 5 張 |
| 每回合消耗 | 1 張（出牌） |
| 回合間補牌 | 不補牌 |
| 總共出牌 | 3 張（3 回合） |
| 結束時剩餘 | 2 張（未使用的） |

**設計意圖：** 5 張手牌、3 回合出牌、剩 2 張。手牌管理 = 選擇。你有質詢、反駁、演說各 1 張，你要決定哪 3 張值得打。你不可能全部使用，這迫使你做出優先級判斷。

### 5.6 牌庫構築

#### 多人模式

每個玩家有一個 **15 張的牌庫**，每局隨機抽 5 張作為手牌。

| 牌庫組成 | 數量 | 說明 |
|---------|------|------|
| 質詢 | 3 | 基礎情報收集工具 |
| 反駁 | 3 | 基礎防禦工具 |
| 演說 | 3 | 聲望回復 |
| 結盟 | 2 | 高風險高回報的信任工具 |
| 爆料 | 1 | 核武器，用一次就沒了 |
| 倒戈 | 1 | 緊急逃生路線 |
| 派系專屬 | 2 | 提供派系特色能力 |

玩家可以透過**賽季獎勵、成就解鎖、排位獎勵**替換牌庫中的基礎牌為精良/史詩/傳奇版本。替換不增加牌數，只提升牌的品質。

#### 單機模式

在 Roguelike 模式中，牌庫構築遵循 STS 模式——每屆國會結束後從獎勵中選牌加入牌庫，並可移除不需要的牌。

## 6. 隱藏議程系統

### 6.1 概念

**隱藏議程是 v2 的核心創新。** 每個玩家在遊戲開始時收到 1 張隱藏議程卡，定義了他的勝利條件。議程在投票結束後揭示。

### 6.2 議程設計原則

> **原則一：部分重疊。** 議程之間不是非此即彼——多個玩家的議程可能部分相容。這製造了「暫時盟友」和「必然敵人」的動態關係。

> **原則二：多元路徑。** 議程的勝利條件不只是「法案 X 通過」，還包括聲望門檻、維度條件、社交條件（如「你至少與 1 人成功結盟」）。

> **原則三：非對稱難度。** 有些議程容易完成但低分，有些困難但高分。玩家需要根據局勢判斷是否使用倒戈換一個更有利的議程。

### 6.3 議程類型（4 人局範例）

#### A. 法案派議程（與法案投票結果直接相關）

| 議程名 | 勝利條件 | 基礎分 |
|--------|---------|--------|
| 📜 法案擁護者·A | 法案版本 A 通過 | 40 分 |
| 📜 法案擁護者·B | 法案版本 B 通過 | 40 分 |
| 📜 法案擁護者·C | 法案版本 C 通過 | 40 分 |

#### B. 條件派議程（法案 + 額外條件）

| 議程名 | 勝利條件 | 基礎分 |
|--------|---------|--------|
| 🎯 穩健派·A | 法案 A 通過 且 你的聲望 ≥ 40 | 55 分 |
| 🎯 穩健派·B | 法案 B 通過 且 你的聲望 ≥ 40 | 55 分 |
| 🏆 人望王 | 任何法案通過，且你是全場聲望最高 | 50 分 |
| 🤝 合縱連橫 | 任何法案通過，且你至少與 2 人成功結盟 | 50 分 |

#### C. 攪局派議程（從混亂中獲益）

| 議程名 | 勝利條件 | 基礎分 |
|--------|---------|--------|
| 🔥 議會縱火犯 | 任何法案以 ≤ 3 票差通過（或流局） | 60 分 |
| 🎭 雙面人 | 你的議程在遊戲結束時從未被任何人正確知道 | 65 分 |
| 💀 政治殺手 | 有至少 2 名玩家的聲望在遊戲結束時 ≤ 15 | 60 分 |

### 6.4 議程分配規則

| 規則 | 說明 |
|------|------|
| 4 人局：3 法案派 + 1 攪局/條件派 | 確保大多數人有明確法案立場，但有 1 人是變數 |
| 6 人局：3 法案派 + 2 條件派 + 1 攪局派 | 增加複雜度 |
| 8 人局：3 法案派（部分重複）+ 3 條件派 + 2 攪局派 | 最大混亂 |
| 同一法案版本最多 2 人支持 | 避免過於明顯的多數壓制 |
| 攪局派至少 1 人 | 確保永遠有「不可預測因素」 |

### 6.5 議程與社交的互動

議程系統讓聊天變得有意義：

- **被質詢後的撒謊**：你的議程是「法案 A 通過」，但被質詢後你在公開聊天中說「我的議程是 B」。這是合法的——質詢只讓質詢者看到真相，你可以在公開場合說謊。
- **結盟後的背叛**：你和某人結盟，共享了議程。你發現他的議程和你衝突。你可以選擇下回合爆料他，或者假裝合作到最後一刻再投反對票。
- **議程推理**：根據某人的出牌模式（他一直在質詢 C 的支持者），你可以推測他可能是 A 或 B 的支持者。

## 7. 四維度系統

### 7.1 概念

保留 v1 的四維度框架（民意、財政、軍事、外交），但在 v2 多人模式中**簡化其角色**：

- **v1：** 四維度是核心遊戲狀態，每張法案卡影響維度，維度觸發事件。
- **v2 多人模式：** 四維度主要體現在**法案版本的設計**和**投票結果**中，不在每張卡牌上。
- **v2 單機模式：** 保留 v1 的完整四維度互動系統。

### 7.2 法案與四維度

每局的法案有 A/B/C 三個版本，每個版本對四維度有不同影響：

**範例：【機器法案】**

| 版本 | 民意 | 財政 | 軍事 | 外交 | 誰受益 |
|------|------|------|------|------|--------|
| A. 禁止機器 | +15 | -10 | 0 | -5 | 工人階級（保守黨傾向） |
| B. 保護財產 | -10 | +15 | +5 | +5 | 資本家（商行傾向） |
| C. 折衷改革 | +5 | +5 | 0 | +5 | 改革派（自由派傾向） |

**多人模式中，** 四維度的變化在單局內主要是**敘事性的**（增加主題沈浸感），但在**排位賽/賽季模式**中，四維度會跨局累積，影響下一局可用的法案池和事件。

### 7.3 四維度（完整系統——單機模式用）

> 完整四維度系統保留 v1 設計，包括：
> - 維度範圍 0-100，初始值 50
> - 維度互動（軍事↑→財政↓ 等）
> - 維度閾值事件
> - 極端值結局觸發
>
> 詳見 [Section 9](#9-單機模式國會-roguelike)。

## 8. 派系系統

### 8.1 設計調整

v1 的三派系剪刀石頭布體系在概念上很好，但在 v2 的多人模式中需要調整角色：

- **v1：** 派系決定你的卡牌池和遊戲風格（防禦/連段/控制）。
- **v2：** 派系決定你的**專屬牌、初始聲望、和被分配議程的傾向性**，但不決定你的出牌策略。同一個保守黨玩家可能因為議程不同而採取完全不同的策略。

### 8.2 三大派系

#### 🏛️ 保守黨（The Royalists）— 穩定者

> _「傳統即力量。穩定即正義。」_

| 屬性 | 值 |
|------|-----|
| 初始聲望 | 60（最高） |
| 專屬牌 | 王室否決、教會庇護 |
| 派系特質 | 防禦向——專屬牌偏向保護自己和盟友 |
| 議程傾向 | 較常被分配到法案 A 或穩健派議程 |

#### 📰 自由派（The Reformists）— 煽動者

> _「人民的聲音，就是法律的力量。」_

| 屬性 | 值 |
|------|-----|
| 初始聲望 | 50（中等） |
| 專屬牌 | 街頭小報、群眾演說 |
| 派系特質 | 情報向——專屬牌偏向獲取或散播資訊 |
| 議程傾向 | 較常被分配到法案 C 或條件派議程 |

#### 💰 商行聯盟（The Merchant Guild）— 操縱者

> _「每個人都有價格。問題只是多少。」_

| 屬性 | 值 |
|------|-----|
| 初始聲望 | 45（最低） |
| 專屬牌 | 收買、偽造文書 |
| 派系特質 | 欺騙向——專屬牌偏向暗中操作和散播假情報 |
| 議程傾向 | 較常被分配到法案 B 或攪局派議程 |

### 8.3 派系平衡

三個派系的平衡不是靠「剪刀石頭布剋制」，而是靠**不同的資訊戰優勢**：

| 派系 | 優勢 | 劣勢 |
|------|------|------|
| 保守黨 | 高起始聲望 = 容錯高，防禦牌讓你更安全 | 情報能力弱，容易被蒙在鼓裡 |
| 自由派 | 情報獲取能力強，能更快看清局勢 | 起始聲望中等，無特殊防禦 |
| 商行聯盟 | 偽造和暗中操作能力強，最擅長欺騙 | 起始聲望最低，被爆料風險高 |

## 9. 單機模式：國會 Roguelike

### 9.1 概念

保留 v1 的框架，核心改動：

1. **卡牌系統換用 v2 的六種行動牌**（但增加單機專屬牌類型：法案卡、事件卡）
2. **AI 議員使用 v2 的出牌邏輯**（同時出牌、有隱藏議程）
3. **四維度系統保留完整版**（維度互動、閾值事件、極端值結局）
4. **遺產系統保留**（每屆通過法案轉化為永久加成）

### 9.2 核心循環

```
選擇派系
    │
    ▼
隨機開局背景（20 種）
    │
    ▼
┌──────────────────────────────┐
│  第 N 屆國會                  │
│  ├─ 暗流：查看 AI 議員陣容     │
│  ├─ 辯論：3 回合同時出牌      │
│  ├─ 投票：法案版本表決         │
│  ├─ 結算：四維度變化 + 事件    │
│  ├─ 遺產轉化                 │
│  └─ 牌庫構築（獎勵選牌）      │
└──────────────────────────────┘
    │
    ▼（重複 5-8 屆）
    │
    ▼
歷史結局觸發
```

### 9.3 單機與多人的差異

| 維度 | 多人模式 | 單機模式 |
|------|---------|---------|
| 對手 | 真人玩家 | AI 議員（3-7 個） |
| 聊天 | 核心機制 | 替換為「AI 對話系統」（AI 會發言、騙你、提議結盟） |
| 牌庫 | 固定 15 張 | 可構築，每屆新增卡牌 |
| 四維度 | 簡化版（敘事用） | 完整版（核心機制） |
| 時長 | 10-12 分鐘 | 20-25 分鐘 |
| 結局 | 勝/負 | 10+ 種歷史結局 |

### 9.4 AI 議員行為

AI 議員使用與真人相同的出牌系統，但有額外的**行為個性**：

| 個性 | 出牌傾向 | 聊天風格 |
|------|---------|---------|
| 激進型 | 多質詢、多爆料 | 主動控訴，經常挑釁 |
| 防守型 | 多反駁、多演說 | 低調，避免衝突 |
| 外交型 | 多結盟 | 主動提議合作 |
| 狡詐型 | 先結盟再背叛 | 友善但不可信 |

### 9.5 難度系統

| 難度 | AI 行為品質 | 隨機因子 | 玩家勝率 |
|------|-----------|---------|---------|
| 議員 | AI 偶爾犯錯，不會主動爆料 | 70% | ~80% |
| 黨鞭 | AI 合理出牌，偶爾結盟後背叛 | 45% | ~60% |
| 首相 | AI 高效使用情報，會協調集火 | 20% | ~40% |
| 攝政王 | AI 近乎完美決策 | 5% | ~20% |

## 10. 結局與計分系統

### 10.1 多人模式計分

```
最終分數 = 議程基礎分（完成 = 滿分 / 未完成 = 0）
         + 剩餘聲望
         + 社交成就加分
```

#### 社交成就加分

| 成就 | 分數 | 條件 |
|------|------|------|
| 🤝 信守承諾 | +10 | 與至少 1 人成功結盟，且全程未對盟友使用爆料 |
| 🗡️ 完美背叛 | +15 | 先結盟後爆料同一人，且最終完成議程 |
| 🎭 影子大師 | +20 | 你的議程在遊戲結束時沒有被任何人（透過質詢/結盟）知道 |

| 🛡️ 金鐘罩 | +10 | 全程未被任何質詢成功偷看議程 |

### 10.2 排名

每局結束後所有玩家按最終分數排名。1-4 名各有不同的排位積分獎勵。

### 10.3 單機模式結局

保留 v1 的多結局系統：10+ 種歷史結局，由四維度狀態和特定事件觸發。

| 結局名 | 觸發條件 | 描述 |

| 🕊️ 和平改革 | 民意 ≥ 80 且軍事 ≤ 30 | 通過和平手段實現改革 |
| 🏴 愛爾蘭獨立 | 外交 ≤ 10 且民意 ≤ 30 | 帝國分崩離析 |

## 11. 配對與房間系統

### 12.2 同時出牌的同步設計

同時出牌是 v2 的核心機制，需要可靠的同步：

```
客戶端 A ──┐                    ┌── 客戶端 A（收到所有牌）
客戶端 B ──┤ 出牌 → 伺服器蒐集  │── 客戶端 B（收到所有牌）
客戶端 C ──┤    全部到齊後      │── 客戶端 C（收到所有牌）
客戶端 D ──┘    同時廣播 ───────┘── 客戶端 D（收到所有牌）
```

**同步規則：**
1. 伺服器等待所有玩家出牌（或超時自動出演說）
2. 全部收齊後，伺服器計算結算結果
3. 同時向所有客戶端廣播：翻牌動畫 → 結算效果
4. 私密資訊（質詢結果）只發給相關玩家

**防作弊：**
- 客戶端不知道其他人出了什麼牌，直到伺服器廣播
- 議程卡資訊只在伺服器端，客戶端只知道自己的
- 質詢/結盟的私密結果透過加密通道發送

## 14. 附錄：v1 → v2 變更對照

| 維度 | v1 | v2 | 變更原因 |
|------|-----|-----|---------|
| 核心動詞 | 攻擊/防禦 | 偵查/隱藏/結盟/背叛 | 政治遊戲的本質是資訊戰，不是數值對轟 |
| 出牌方式 | 回合制輪流 | 同時出牌 | 消除等待時間，增加翻牌張力 |
| 等待時間 | ~75% | 0% | 所有人同時決策 |
| 聲望角色 | 生命值（歸零淘汰） | 三合一資源（投票權重+行動成本+計分） | 避免被集火淘汰的糟糕體驗 |
| 社交層 | 聊天是附加品 | 聊天是核心機制（撒謊/讀人） | 社交推理遊戲的社交必須有意義 |
| 勝利條件 | 聲望最高者/陣營勝利 | 完成隱藏議程 | 每人有不同目標，增加博弈複雜度 |
| 卡牌分類 | 法案/策略/人脈/陰謀/交易 | 質詢/反駁/演說/結盟/爆料/倒戈 | 從功能分類改為政治行動分類 |
| 派系角色 | 決定卡牌池和戰鬥風格 | 決定專屬牌和議程傾向 | 派系提供特色但不限制策略 |
| 單局時長 | 10-15 分鐘 | 10-12 分鐘 | 更精煉 |
| 卡牌稀有度 | 強度遞增 | 策略彈性遞增 | 避免 P2W 感受 |
